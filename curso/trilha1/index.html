🧩 O que é uma Agent Skill (e por que o formato venceu)

Da definição de uma skill ao motivo de o formato SKILL.md ter virado padrão de fato — e como o skills.sh virou o "npm de skills".

O que é:

Uma skill é um arquivo SKILL.md — frontmatter YAML (name + description) mais um corpo Markdown — que descreve um comportamento. O agente lê e incorpora essa instrução ao seu próprio raciocínio.

Por que aprender:

É a unidade atômica de todo o ecossistema. Sem entender o que é um SKILL.md, o resto (catálogo, install count, grupos) não faz sentido.

Conceitos-chave:

SKILL.md · frontmatter YAML · name + description · corpo Markdown

O que é:

Skill não é código que roda. É texto que o agente lê e segue quando o contexto da conversa bate com a description. A ativação é automática e por relevância, não por chamada explícita.

Por que aprender:

Essa diferença muda como você escreve. Você não programa passos — você descreve quando e o porquê, e deixa o agente decidir como aplicar.

Conceitos-chave:

ativação por contexto · description como gatilho · sem execução direta

O que é:

SKILL.md é texto puro: roda em 50+ agentes, versiona via git como qualquer arquivo de repo, e já soma 53,9M de instalações no agregado do skills.sh. Markdown venceu por ser o menor denominador comum.

Por que aprender:

Entender por que o formato pegou explica por que vale investir nele em vez de prender conhecimento em ferramentas proprietárias.

Conceitos-chave:

portabilidade 50+ agentes · versionamento git · 53,9M installs

O que é:

São camadas distintas: prompt é efêmero; CLAUDE.md é contexto fixo do projeto; MCP dá ferramentas/dados; subagente é um executor isolado; a skill é conhecimento reutilizável que dispara por contexto.

Por que aprender:

Saber escolher a camada certa evita transformar tudo em skill (ou nada). Cada problema tem o mecanismo adequado.

Conceitos-chave:

prompt efêmero · CLAUDE.md sempre ativo · MCP = ferramentas · subagente = execução · skill = conhecimento sob demanda

O que é:

O skills.sh é o catálogo público: indexa skills por repo, mostra install count (sinal de qualidade) e instala com npx skills add owner/repo. É o npm do mundo de skills.

Por que aprender:

É onde você descobre o que já existe antes de criar, e onde você publica para outros instalarem.

Conceitos-chave:

catálogo · install count · npx skills add · leaderboard

O que é:

Publicam grandes players — vercel-labs, anthropics, microsoft — e milhares da comunidade, totalizando 5.075 repos fonte. Não é um jardim murado de uma empresa só.

Por que aprender:

Saber quem domina e quem ainda não chegou mostra onde há espaço para a sua skill.

Conceitos-chave:

vercel-labs · anthropics · microsoft · comunidade · 5.075 repos

🗺️ O mapa do skills.sh: 39k skills, a lei de potência, os 16 grupos

O tamanho real do catálogo, a curva brutal de instalações e os 16 grupos temáticos — e o que tudo isso te ensina antes de criar a sua skill.

O que é:

O skills.sh cataloga 39.366 skills vindas de 5.075 repos fonte, somando 53,9M de instalações distribuídas em 16 grupos temáticos.

Por que aprender:

O número total dá a escala: o ecossistema é grande o bastante para já ter quase tudo — e específico o bastante para ainda ter buracos.

Conceitos-chave:

39.366 skills · 5.075 repos · 53,9M installs · 16 grupos

O que é:

A distribuição é uma lei de potência: 60,2% das skills têm menos de 100 installs, e apenas 0,3% (131 skills) passam de 100k. A maioria vive na cauda longa.

Por que aprender:

Te tira da ilusão de que "publicar = sucesso". A esmagadora maioria das skills quase não é instalada.

Conceitos-chave:

lei de potência · cauda longa · 60,2% <100 · 0,3% = 131 skills 100k+

O que é:

As 100 skills mais instaladas concentram 43,7% de todas as instalações. Quase metade do bolo está em 100 nomes, entre quase 40 mil.

Por que aprender:

Mostra o efeito "vencedor leva quase tudo" — e por que competir de frente com as líderes é difícil.

Conceitos-chave:

top 100 = 43,7% · efeito de rede · winner-takes-most

O que é:

O catálogo se divide em 16 grupos. Por contagem lidera AI/LLM/Agents (12.693 skills, 12,9M installs); por instalação por skill brilha DevOps&Cloud (907 skills, 8,3M installs). Web&Frontend tem 4.694 skills.

Por que aprender:

Ver volume vs instalação por grupo revela onde há muita oferta e onde poucas skills capturam muita demanda.

Conceitos-chave:

AI/Agents 12.693 · Web 4.694 · DevOps 8,3M · Backend 2.685 · Testing 1.339

O que é:

Não é só skill que concentra — fonte também. Poucos repos (vercel-labs, anthropics, microsoft, larksuite) respondem por uma fatia gigante das instalações, mesmo entre 5.075 repos.

Por que aprender:

Marca e distribuição contam. Uma skill ótima num repo desconhecido começa em desvantagem.

Conceitos-chave:

vercel-labs · anthropics · microsoft · larksuite · marca + distribuição

O que é:

A lição prática: descubra o que já existe antes de criar; mire nichos vazios em vez de duelar com líderes; e posicione sua skill do lado certo da curva (resolvendo dor real, não só repetindo o que já tem).

Por que aprender:

É a ponte da Trilha 1 para o resto do curso: o mapa orienta onde e como criar.

Conceitos-chave:

pesquisar antes de criar · nicho vazio · lado certo da curva
